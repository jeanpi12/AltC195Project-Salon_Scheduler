AltC195Project Javadoc Index

This document compiles all Javadoc comments following Oracle’s best-practices guidelines. Each class and its members are documented here, and methods using lambda expressions include justifications.

Class: AddClients

Purpose: Controller for the “Add Client” screen; populates country list and handles client creation.

Methods

initialize(URL url, ResourceBundle rb)
Populate the country ComboBox from the database and set default values for new client form.
actionSaveButton(ActionEvent event)
Validates input fields and inserts a new client via ClientDAO.addClient. Shows confirmation or error alert.
actionCancelButton(ActionEvent event)
Abandons client creation and returns to the main Clients screen.

Class: AddNewAppointments

Purpose: Controller for the “Add Appointment” screen; initializes form controls, enforces business rules, and persists new appointments.

Methods

initialize(URL url, ResourceBundle rb)
Sets up date pickers, time slots, stylist and client ComboBoxes, and disables past dates.
isWithinBusiness(LocalDateTime start, LocalDateTime end)
Checks that proposed times fall within salon business hours and not on holidays.
actionSaveButton(ActionEvent e)
Gathers user input, validates business hours and overlaps, then calls AppointmentDAO.addingAppointments. Throws alert on invalid data.
actionCancelButton(ActionEvent e)
Returns to the Appointments list without saving.

Class: Appointments

Purpose: Controller for the main Appointments view; displays, filters, and dispatches CRUD operations on appointments.

Methods

initialize(URL url, ResourceBundle rb)
Loads all appointments into the table and sets up radio‐button filters.
allAppointments(ActionEvent e)
Displays every appointment.
weekAppointments(ActionEvent e)
Displays appointments for the current week.
monthAppointments(ActionEvent e)
Displays appointments for the current month.
actionAppointmentAdd(ActionEvent e)
Switches to the AddNewAppointments view.
actionAppointmentUpdate(ActionEvent e)
Selects an appointment and switches to ModifyAppointments.
actionAppointmentDelete(ActionEvent e)
Deletes the selected appointment after user confirmation.
backMainMenu(ActionEvent e)
Returns to the main menu.

Class: Clients

Purpose: Controller for the main Clients view; displays, filters, and dispatches CRUD operations on clients.

Methods

initialize(URL url, ResourceBundle rb)
Loads all clients into the table. Uses: allClients.forEach(c -> countryCombo.getItems().add(c.getCountry()));
Justification: Lambda forEach simplifies populating the country ComboBox from the client list without explicit loops.
actionClientAdd(ActionEvent e)
Switches to the AddClients view.
actionClientUpdate(ActionEvent e)
Selects a client and switches to ModifyClients view.
actionClientDelete(ActionEvent e)
Inactivates or deletes the selected client after confirmation, cleans up related appointments first.
backMainMenu(ActionEvent e)
Returns to the main menu.

Class: ModifyAppointments

Purpose: Controller for the “Modify Appointment” screen; reuses AddNewAppointments init logic, pre-fills fields, and updates appointments.

Calls AddNewAppointments.initialize(), then populates form fields from selected appointment.

Class: ModifyClients

Purpose: Controller for the “Modify Client” screen; pre-fills client data and updates via ClientDAO.updateClient.

Methods

initialize(URL url, ResourceBundle rb)
Populates country list, loads selectedClient data, and sets active‐checkbox state.
actionSaveButton(ActionEvent e)
Validates inputs, deletes related appointments if inactivating, updates client record, then returns to Clients view.
actionCancelButton(ActionEvent e)
Cancels edit and returns to Clients view.

Class: Reports

Purpose: Controller for the Reports view; generates summary reports on appointments and clients.

Methods

generateClientReport()
Filters only active clients: clients.stream().filter(c -> c.getClientActivity() == 1).collect(Collectors.toList())
Justification: Using streams and lambdas makes filtering concise and readable compared to manual loops.
generateAppointmentReport()
Aggregates appointment counts by type, week, or month and displays in a chart.
